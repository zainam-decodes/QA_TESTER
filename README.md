# SauceDemo Manual Testing Project

## Project Overview

This project focuses on performing manual testing on the SauceDemo E-Commerce web application using multiple test user accounts. The objective was to identify functional, UI, navigation, validation, and performance-related defects and document them professionally using QA documentation practices.

## Key Bugs Identified

* Checkout accepts invalid pincode characters
* Reset App State not functioning
* Product price inconsistency
* Product images replaced with dog images
* Product filter/sorting not functioning
* Last Name field not accepting input
* Finish button not functioning
* Cart icon alignment issues
* Slow UI performance

The SauceDemo application contains multiple functional, UI, validation, and navigation-related defects across different user profiles. Critical issues were identified in checkout validation, product sorting, and cart functionality. The project demonstrates practical experience in manual testing, defect reporting, and QA documentation.
